Workflow Feature Flags
Enable or disable specialized workflows for this tenant's operations
{{ form.use_strict_sales_workflow }}

Enforces a 3-step sales process:
Attendant creates invoice → Cashier collects payment & prints receipt → Manager dispatches stock.
Without this, any authorised role can complete a sale in one step.

{{ form.receipt_print_copies }}

Number of identical receipt copies to print upon successful payment. (e.g. 3 copies: Customer, Manager, Cashier).

{{ form.require_customer_on_invoice }}

When enabled, all sales/invoices must have a customer name and phone number recorded. Walk-in customer details will still be captured but not saved as a credit customer.

{{ form.use_bulk_inventory_receiving }}

Enables the Goods Receipt module, allowing shop managers to receive multiple products from a supplier in a single transaction instead of item by item. Replaces the need for a Central Stores location.

{{ form.require_accountant_for_bulk_receiving }}

If enabled, bulk Goods Receipts submitted by a shop manager will sit in Pending Approval status until an Accountant or Admin logs in to verify and finalize the stock into inventory. Requires "Bulk Inventory Receiving" above.

{{ form.stock_adjustment_approver }}

Defines who must approve pending stock adjustments:
• Manager/Admin — Shop manager of that location or Admin.
• Auditor Only — Only users with the Auditor role.
• Both — Both the manager AND an Auditor must approve.
